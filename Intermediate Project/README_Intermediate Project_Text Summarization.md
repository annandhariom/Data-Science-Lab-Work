# NLP Text Summarization 

> **Type:** Intermediate NLP Project  
> **Task:** Automatic text summarization using extractive and abstractive approaches  
> **Models:** TF-IDF Extractive Baseline · BART Transformer (Abstractive)  
> **Evaluation:** ROUGE-1, ROUGE-2, ROUGE-L  

---

## Project Overview

This project builds an end-to-end NLP pipeline for **automatic text summarization** of news articles. Given a long-form article, the system produces a concise summary that captures the core information.

Two distinct approaches are implemented and compared:

| Approach | Model | Description |
|----------|-------|-------------|
| Extractive | TF-IDF Sentence Scoring | Scores sentences by term importance, extracts top-k |
| Abstractive | BART (`facebook/bart-large-cnn`) | Generates new text using a pre-trained seq2seq transformer |

## Methodology

### Extractive Summarization (TF-IDF)

**Algorithm:**
1. Tokenize the article into sentences using NLTK's `sent_tokenize`.
2. Clean each sentence (lowercase, remove special characters).
3. Build a TF-IDF matrix where each sentence is treated as a document.
4. Score each sentence as the sum of its TF-IDF weights; penalize very short sentences.
5. Select the top-3 scoring sentences, preserving their original order.

**Pros:** No training required, fast, interpretable.  
**Cons:** Cannot paraphrase; limited to extracting existing sentences.

### Abstractive Summarization (BART)

BART (Bidirectional and Auto-Regressive Transformers) is a sequence-to-sequence model with a BERT-like encoder and a GPT-like decoder. `facebook/bart-large-cnn` is pre-trained on a large text corpus and then fine-tuned on the CNN/DailyMail news summarization dataset.

**Generation parameters:**
```python
max_length=130          # maximum output tokens
min_length=30           # minimum output tokens
num_beams=4             # beam search width
length_penalty=2.0      # encourages longer, complete summaries
no_repeat_ngram_size=3  # prevents repetitive phrases
```

**Pros:** Generates novel, fluent text; typically more concise than extractive.  
**Cons:** Computationally expensive; can occasionally hallucinate facts.

## Evaluation: ROUGE Metrics

| Metric | Measures |
|--------|----------|
| ROUGE-1 | Unigram (single word) overlap between generated and reference summary |
| ROUGE-2 | Bigram (two-word phrase) overlap |
| ROUGE-L | Longest Common Subsequence — captures sentence-level structural similarity |

Each metric produces Precision, Recall, and **F1 (reported)**. Stemming is enabled for fairer comparison.

### Typical Results

| Model | ROUGE-1 F1 | ROUGE-2 F1 | ROUGE-L F1 |
|-------|-----------|-----------|-----------|
| TF-IDF Extractive | ~0.40 | ~0.16 | ~0.28 |
| BART Abstractive | ~0.52 | ~0.24 | ~0.38 |

---

## Key Insights

1. **BART outperforms TF-IDF** on all ROUGE metrics, especially ROUGE-2, because it captures phrase-level semantics rather than just lifting existing sentences.
2. **TF-IDF summaries are longer** — extracting 3 full sentences overshoots the ~40-word reference target.
3. **Science and Environment articles** are hardest to summarize due to dense technical language and many specific facts.
4. **The compression ratio target is ~0.20** — a good summary should be ~20% the length of the article.
5. **ROUGE has limits** — a model can produce a semantically accurate summary that still scores low if it uses different vocabulary from the reference.
